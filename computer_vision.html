Computer Vision

Aan de hand van de reader HCI Technologies 2019 - Processing & Computer Vision heb ik de volgende opdrachten in Processing gemaakt. Deze opdrachten geven basis kennis en ervaring over Processing. Processing is een open source programmeertaal dat in de Java omgeving draait. De programmeertaal is bijna gelijk aan Java, maar het is makkelijker en sneller om te gebruiken. Het is een omgeving voor mensen die afbeeldingen, animaties en interacties willen programmeren.

Opdracht 1




De voorwaarden voor deze opdracht waren:


  • Minimaal drie ellipsen
  • Ellipsen op verschilende locaties
  • Elipsen met verschillende kleuren
  • Verschillende breedtes en hoogtes
  • Met en zonder randen

Ik heb een programma geprogrammerd een groene ellips zonder rand, een zwarte ellipse met groene rand en een witte ellipse met groene rand weergeeft. Deze ellipsen staan op verschillende locaties.

Opdracht 2




De voorwaarden voor deze opdracht waren:


  • Minimaal vier verschillende vormen (geen cirkels)
  • De vormen in verschillende kleuren
  • De vormen op verschillende locaties

Ik heb een programma geprogrammeerd dat een groene quad zonder rand, een blauw vierkant met een witte rand, een witte dikke lijn en een oranje driehoek weergeeft.

Opdracht 3


De voorwaarden voor deze opdracht waren:


  • Een programma dat tenminste twee verschillende vormen
  • De vormen moeten verschillende kleuren hebben
  • Het programma moet interactief zijn

Ik heb een programma geprogrammeerd waarbij een gele, blauwe en groene cirkel met witte randen vanzelf groter worden. Een oranje cirkel volgt de muis.

Opdracht cv1



De opdracht was het bekijken van de LiveCamTest code en ontdekken welke regel ervoor zorgt dat een vierkant rond een gezicht verschijnt. De code moest aangepast worden zodat er een cirkel in plaats van een vierkand rond een gezicht verschijnt.


Ik heb de code die geel gehighlight is aangepast.

Opdracht cv2



De opdracht was het aanpassen van de code zodat er bij elk gezicht een "anonieme" zwarte balk voor de ogen verschijnt.

Opdracht cv3

De opdracht was om de code van OpenCVMultipleColorTrackingLiveCam aan te passen zodat er iets "cools" gebeurd

Ik heb de code op regel 171 van mousePressed naar mouseMoved aangepast en de code op regel 202 van keyPressed, keyReleased gemaakt.

In plaats van het ingedrukt houden van een cijfer en het klikken op een kleur. Heeft de verandering in de code ervoor gezorgd dat de kleuren na het klikken op nummers 1-4 bij hover meteen getrackt worden. Als er dan op een ander nummer tussen 1 en 4 wordt geklikt, blijft het laatst gehoverde kleur staan bij nummer 1.
